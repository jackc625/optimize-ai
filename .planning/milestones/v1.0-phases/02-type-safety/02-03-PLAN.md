```typescript
const handleRecalculate = async () => {
  setIsRecalculating(true);
  try {
    const [result] = await Promise.all([
      refetch(),
      new Promise((r) => setTimeout(r, 400)),
    ]);
    if (result.error) {
      toast.error("Failed to recalculate macros.");
    } else {
      toast.success("Macros recalculated!");
    }
  } catch {
    toast.error("Failed to recalculate macros.");
  } finally {
    setIsRecalculating(false);
  }
};
```

Key details:
- `Promise.all` with a 400ms minimum delay ensures the loading state ("Recalculating...") is visible to the user, even when the refetch completes in ~50ms.
- `refetch()` returns `{ data, error, ... }` — check `result.error` to distinguish success from failure without throwing.
- The outer `catch` handles unexpected exceptions (network failures, auth expiry).
- `finally` guarantees `setIsRecalculating(false)` always runs — the button never gets stuck in a disabled state.
- `toast` is already imported on line 14 — no new imports needed.
- Do NOT modify any other part of the component. Only replace the handleRecalculate function body.
